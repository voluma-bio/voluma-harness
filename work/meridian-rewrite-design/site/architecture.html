System Architecture

One local daemon is the authority for every Voluma surface. The CLI, recursive subagents, MCP write tool, and web app connect through it rather than creating competing sources of truth.

Product Surfaces

Surface Caller Primary job Process boundary
CLI Human, Claude Code, subagents, prompts Start sessions; spawn, wait, and cancel; run work, session, context, mars, and write commands CLI → daemon RPC
Subagent spawning Recursive voluma spawn from agents Create child threads with bounded capability Parent → CLI → daemon → harness
Webapp Human browser/mobile Live editor, thread UI, spawn dashboard, event, token, and cost views Browser → HTTP/WS daemon

Dependency Direction

Stable dependency direction: apps → packages → shared flow packages. Shared flow packages are library dependencies, not a forked app base.
