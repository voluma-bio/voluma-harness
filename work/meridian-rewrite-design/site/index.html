Voluma v1

Local-first agent execution platform for researchers and developers. Unifies meridian-cli (Python), meridian-flow (cloud CRDT), and the existing Voluma server into one architecture: local by default, org-hosted when teams need it.

v1 replaces three separate implementations with one local TypeScript authority process backed by SQLite/Postgres, Yjs, and shared @voluma/* packages. Agents call a shared write capability; every write lands in the same Yjs document authority that the web editor uses. Deployable locally (researcher workstation) or hosted (org-managed Postgres + Daytona sandbox).

Key decisions

Local-first, data sovereign
Runs on the researcher's machine. No cloud dependency. Org-hosted deployment optional — same architecture, different DB/sandbox adapters.
SQLite/Postgres via Drizzle
SQLite WAL locally, Postgres hosted. Same schema. Crash-only spirit preserved via transactional writes and idempotent recovery.
Two harness classes
Claude Code native (subprocess, stream-json) + Unified API (in-process, Anthropic/OpenAI/Gemini). No more five adapters.
Daemon owns all spawns
Skills/config resolve daemon-side. 3-level workspace resolution replaces 6-level task-dir cascade.
Write as the seam
MCP for Claude Code primary, bash CLI for subagents (local or Daytona sandbox), in-process for API agents. Same WriteService, same Yjs doc.
Dual sandbox
OS sandbox (bwrap/seatbelt) for local. Daytona containers for hosted. Same capability model, same process containment interface.
